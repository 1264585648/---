# 第 7 页：第三类的法律风险与已有执法信号

## 本页要讲清楚的一句话

即使公开资料中未必已经出现大量“AI API 中转站”名义的判决，第三类中转所依赖的上游行为——非法获取系统数据、撞库、盗刷信用卡、侵犯个人信息、帮助信息网络犯罪——早已有成熟的刑事规制和公开案例。

## 内容概要

这一页承接第 6 页，重点讲法律风险。

第三类中转站不是一个单独的新罪名，但它可能同时踩到多个既有法律风险区：

- 非法获取计算机信息系统数据：撞号、撞 API Key、非法获取账号凭证、突破接口限制获取数据。
- 非法控制计算机信息系统：通过技术手段让他人系统接受非授权指令，或者绕开官方客户端控制数据交互。
- 信用卡诈骗：盗刷信用卡、使用非法支付工具薅取订阅或额度。
- 侵犯公民个人信息：购买、收集、转卖账号资料、手机号、身份认证信息、支付信息。
- 帮助信息网络犯罪活动：为上游黑产提供账号、支付、服务器、转发、结算等帮助。
- 生成式 AI 服务合规责任：面向公众提供生成式 AI 服务，可能涉及内容安全、数据保护和服务提供者责任。

公开案例方面，谨慎表述为“相关行为已有判例和执法信号”：

- 最高检曾发布“撞库”相关案例：行为人利用撞库程序和身份认证信息，非法获取网站身份认证信息，被作为非法获取计算机信息系统数据案件处理。
- 最高检相关文章和案例强调，突破接口加密、伪造请求、绕过官方客户端获取非公开数据，可能构成非法获取计算机信息系统数据或非法控制计算机信息系统。
- 多地检察机关披露过盗刷银行卡、信用卡诈骗、洗钱、帮助信息网络犯罪等案件，说明“盗刷支付能力再变现”的链条已经被刑法规制。

## 关键论点

- “AI API 中转”可能是新包装，但上游违法行为不是新问题。
- 撞号、撞 Key、盗刷、凭证交易、非法接口调用，都可能落入既有刑事罪名。
- 只要中转站明知或应知额度来源非法，仍继续销售、结算或提供技术帮助，风险会从上游传导到平台和参与者。
- 对下游用户来说，接入脏额度也可能带来服务中断、数据泄露、资金损失和协查风险。

## 口播草稿

这里要说明一点：目前公开资料里，不一定已经有大量以“AI API 中转站”为名义的判决。

但这不代表它没有法律风险。因为第三类中转依赖的上游行为，本来就已经被成熟规制了。

比如撞号、撞 API Key，可能涉及非法获取计算机信息系统数据。  
盗刷信用卡、用非法支付能力薅订阅，可能涉及信用卡诈骗。  
购买、收集、转卖账号资料和身份信息，可能涉及侵犯公民个人信息。  
如果有人为这些链条提供服务器、账号、支付、转发、结算，还可能触碰帮助信息网络犯罪活动。

公开案例里，最高检曾提到过撞库案件，也有非法爬取、破解接口、绕过官方客户端获取非公开数据的案例。盗刷银行卡和信用卡诈骗，更是已经有大量办案案例。

所以第三类中转并不是法律空白。  
它只是把老问题换了一个 AI API 的外壳。

对用户来说，最现实的风险也很简单：服务随时断、余额可能没、数据可能泄露，严重时还可能卷入协查。

## 画面重点

### 元素 1：法律风险矩阵

```text
撞号 / 撞 API Key -> 非法获取计算机信息系统数据
盗刷信用卡 -> 信用卡诈骗
账号资料交易 -> 侵犯公民个人信息
服务器 / 支付 / 结算帮助 -> 帮信风险
面向公众提供 AI 服务 -> 生成式 AI 服务合规责任
```

### 元素 2：公开执法信号

```text
撞库案件
非法获取系统数据
越界爬虫 / 破解接口
盗刷银行卡
信用卡诈骗与洗钱
```

### 元素 3：结论卡

```text
不是法律空白，
只是旧风险套上了 AI API 外壳。
```
