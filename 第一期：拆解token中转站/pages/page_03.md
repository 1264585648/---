# 第 3 页：三类中转站概览

## 本页要讲清楚的一句话

中转站不是单一物种，而是一条从相对合规、到灰色套利、再到违法犯罪的风险光谱；判断一家平台靠不靠谱，先要看它属于哪一类。

## 内容概要

这一页做概览式介绍，把中转站分成三类。

第一类是相对合规的 API 中转或企业网关。它们通常有明确授权、企业采购、正规账单和服务协议，本质是帮助用户统一接入多个模型、管理成本、做权限和审计。

第二类是订阅转 API 的灰色地带。它们常见做法是把原本面向个人或特定场景的订阅额度，包装成 API 能力对外售卖。它不一定一开始就表现为黑产，但经常处在平台规则、授权边界和商业使用限制的灰区。

第三类是几乎零成本的违法来源。比如白嫖账号、复用凭证、API Key 撞号、盗刷信用卡等。这类不是“商业模式创新”，而是把非法获取的额度转卖成低价 API，风险最高。

## 关键论点

- 不能把所有中转站混为一谈。
- 合规 API 网关的价值是统一接入、管理和审计。
- 订阅转 API 属于灰色地带，核心问题是授权边界和平台规则。
- 白嫖账号、凭证复用、API Key 撞号、盗刷信用卡等属于明显高风险甚至违法行为。
- 价格越低、来源越不透明，越可能往风险光谱右侧移动。

## 口播草稿

讲到这里，要先把中转站分成三类，不然很容易一概而论。

第一类，是相对合规的 API 中转，或者叫企业级 AI 网关。  
它们有明确的上游授权、采购合同、账单和服务协议，主要价值是帮企业统一接入多个模型，做权限、成本、日志和审计管理。

第二类，是订阅转 API 的灰色地带。  
比如把原本面向个人使用、网页使用或特定场景的订阅额度，重新包装成 API 服务卖给别人。  
这类问题不一定一眼就是黑产，但它往往踩在平台规则、授权边界和商业使用限制的边缘。

第三类，就不是灰色了，而是违法风险。  
比如白嫖账号、复用凭证、API Key 撞号、盗刷信用卡，把几乎零成本甚至非法获取的额度拿出来转卖。

所以你看到一个低价 API 时，不能只看价格。  
真正要问的是：它的上游来源是什么？授权是否真实？如果出问题，责任由谁承担？
