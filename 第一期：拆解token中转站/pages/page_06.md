# 第 6 页：第三类，几乎零成本的违法来源

## 本页要讲清楚的一句话

第三类中转站的危险在于，它不是在做“低价采购”，而是在把非法获取的账号、凭证、额度或支付能力，包装成低价 API 对外转卖。

## 内容概要

这一页介绍最需要重点警惕的一类：几乎零成本的违法来源。

它和第二类灰色地带的区别是，第二类至少还可能来自真实订阅额度，而第三类直接把来源建立在虚假注册、凭证滥用、撞号、盗刷和非法获取数据上。

常见风险类型包括：

- 注册机批量获取账号：通过自动化注册、批量手机号、接码、虚假身份等方式制造账号资源。
- 虚拟银行卡白嫖订阅：利用虚拟卡、黑卡、拒付或支付风控漏洞薅取订阅额度。
- 撞号和撞 Key：利用泄露账号、弱口令、泄露 API Key 或自动化尝试，非法获取他人访问权限。
- 网页转 API：把原本只允许网页端使用的访问能力，包装成对外 API 服务。
- 凭证复用：把 Cookie、Session、Token、账号授权等凭证作为可复用资源转卖或共享。

这一页不讲操作方法，只讲风险结构：这些来源越接近“零成本”，越可能意味着成本被转嫁给被盗刷者、平台、真实账号持有人和下游用户。

## 关键论点

- 几乎零成本并不是商业优势，而是高风险信号。
- 账号、凭证、支付工具、API Key 都可能成为被滥用的“访问设备”或数字资产。
- 网页转 API 和凭证复用会绕开原始产品的使用边界，带来风控和责任问题。
- 下游用户买到的不是稳定服务，而是随时可能被封禁、追责或断供的脏额度。

## 口播草稿

第三类，是最需要重点警惕的：几乎零成本的违法来源。

这类中转站不是靠规模采购，也不是靠企业折扣，而是把非法获得的账号、凭证、额度和支付能力，包装成低价 API。

比如注册机批量获取账号，虚拟银行卡白嫖订阅，撞号，撞 API Key，或者把网页端能力强行转成 API 对外卖。

这些词听起来像技术手段，但本质上都是在绕过授权边界。  
账号不是你的，凭证不是你的，信用卡不是你的，额度也不是你的。

为什么它能便宜？因为真实成本被转嫁给了别人：被盗刷的持卡人、被撞号的账号主人、被滥用的平台，以及最后接入这些脏额度的下游用户。

所以第三类不是“更激进的套利”，而是已经进入违法犯罪风险区。

