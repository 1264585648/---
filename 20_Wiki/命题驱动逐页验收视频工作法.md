# 命题驱动逐页验收视频工作法

## 结论

对于基于长文、研究稿或资料堆栈的视频项目，最稳妥的制作方式不是一次性完成全片，而是：

`命题 -> 大纲 -> 每页内容概要 -> 每页口播稿 -> 全部验收 -> 动效规划 -> 配音规划 -> 制作与渲染`

这个顺序能把返工成本前置消化，避免文案未定就过早锁定动效和配音。

## 适用场景

- 长文改视频。
- 研究报告改科普视频。
- 需要逐页确认的系列视频。
- 需要先确认叙事和事实，再做视觉包装的项目。

## 核心原则

### 1. 先命题，不先做片

先回答四件事：

- 这条视频要回答什么问题？
- 核心观点是什么？
- 给谁看？
- 预期时长和输出形式是什么？

命题清楚，后续的大纲、脚本和分镜才有锚点。

### 2. 先大纲，再逐页内容

不要直接把长文压缩成一条口播。

更好的方法是把视频拆成页级单元，每一页只承担一个表达任务。

每页需要单独确认：

- 标题。
- 本页要讲清楚的一句话。
- 关键论点。
- 口播草稿。
- 审核状态。

### 3. 内容确认后，再做动效和配音

动效、字幕、配音都依赖文案节奏。

如果文案未锁定就提前规划视觉，后面每次改字都会引发重做。

因此建议：

- 先确认内容。
- 再规划每页的视觉类型。
- 最后统一进入配音和渲染。

## 推荐验收格式

```yaml
page: 03
title: "Python 与 TypeScript 的分工"
purpose: "解释为什么视频生产需要双技术栈"
key_points:
  - "Python 负责科学计算和精确推演"
  - "TypeScript 负责 Web 动效和动态排版"
draft_narration: "..."
review_status: "pending"
```

## 经验收益

- 降低返工。
- 保持叙事一致性。
- 让内容审核和视觉制作解耦。
- 适合和 AI 协作分阶段推进。
