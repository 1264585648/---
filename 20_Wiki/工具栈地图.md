# 工具栈地图

## 推荐分工

| 环节 | 首选 | 备选 | 适用场景 |
| --- | --- | --- | --- |
| 数学与科学动画 | Manim | Python + Matplotlib | 公式推导、几何、物理、算法演示 |
| Web 动效与排版 | Remotion / HTML 动画 | Motion Canvas | 字幕动效、图表、界面解说、批量模板 |
| 录屏与剪辑 | OBS + 剪辑软件 | FFmpeg | 软件教程、CLI 演示、快速拼接 |
| 自动合成 | FFmpeg | MoviePy | 转码、拼接、混音、字幕烧录 |
| 配音 | MiniMax | 真人录音 / 其他 TTS | 正式旁白、样音、多版本语气测试 |
| 字幕 | Whisper / ASR | 手工校对 | 自动对齐、短视频动态字幕 |

## 当前判断

对科普和技术解释视频，工具栈不应追求单一大一统。更稳的方式是：
- 用 Markdown 和项目卡管理内容结构。
- 用 Manim 处理严谨动画片段。
- 用 Web 技术处理排版、字幕和现代动效。
- 用 FFmpeg 做最后合成。
- 配音优先使用 MiniMax；在内容未全部验收前，只生成样音，不生成最终旁白。
