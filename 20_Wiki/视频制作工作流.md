# 视频制作工作流

## 一句话流程

选题 -> 资料 -> 观点结构 -> 口播脚本 -> 分镜 -> 资产 -> 渲染/剪辑 -> 配音字幕 -> 质检 -> 发布 -> 复盘。

## 命题驱动模式

当输入是长文、研究报告或资料包时，建议改成更细的逐页验收流程：

命题 -> 大纲 -> 每页内容概要 -> 每页口播稿 -> 全部验收 -> 动效规划 -> 配音规划 -> 制作与渲染。

这个模式适合需要先确认事实与叙事，再决定视觉包装的项目。它能显著减少后期改字带来的返工。

## 标准阶段

### 1. 选题与定位

输出：
- 目标观众
- 视频时长
- 平台
- 核心问题
- 观众看完后应该记住的一句话

### 2. 资料整理

输出：
- 来源列表
- 可信度判断
- 可视化素材候选
- 需要避免的过时信息

### 3. 观点结构

输出：
- 开场钩子
- 三到五个主段落
- 转场逻辑
- 结尾行动或总结

### 4. 口播脚本

要求：
- 口语化，不直接朗读研究报告。
- 每 15-30 秒有一个明确的信息推进。
- 专有名词首次出现要解释。
- 避免把引用链接塞进口播。
- 对于长文改视频，先按页写概要和口播草稿，再整合成全片脚本。

### 5. 分镜

每个镜头记录：
- 时长
- 画面
- 旁白
- 字幕重点
- 素材需求
- 渲染方式

对于页级内容，建议先给每页一个“内容概要 + 验收状态”，确认后再进入镜头级分镜。

### 6. 资产制作

资产包括：
- 图表
- 动画片段
- 截图
- 录屏
- 配音
- 字幕
- BGM 和音效

### 7. 合成与质检

检查：
- 事实是否准确
- 字幕是否可读
- 音量是否稳定
- 节奏是否拖沓
- 画面是否服务解释
- 导出参数是否符合平台要求

### 8. 发布与复盘

记录：
- 标题
- 封面
- 发布时间
- 平台数据
- 评论反馈
- 下次改进
